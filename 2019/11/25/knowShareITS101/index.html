Java Introduction

Java擁有跨平台、物件導向、泛型特性，相比C++優秀繼承物件導向核心，移除複雜得多載與多重繼承，新增垃圾回收器功能。相較於C++來說Java算是簡單又有物件導向可跨平台的程式語言。

物件導向簡單說就是，將其定義類型集合起來就稱為「物件」，有點像在寄送包裹時，賣家會將你所有的訂單大包寄出，相比逐件寄來得更方便。

跨平台性: 在執行Java是透過編譯後，在讓硬體執行，其特性「一次編譯，到處執行」。

想當然裝置勢必有媒介，所以就需要安裝Java執行環境JRE(Java Runtime Environment) 與Java虛擬機器JVM(Java Virtual Machine)，環境與模擬器就能讓裝置run程式。

Spring

Spring由Rod Johnson寫了一本名為 Expert One-on-One J2EE開始，後續更佳進化，話不多說，先來談談3大核心，分別是DI(依賴注入)，IOC(控制反轉)，AOP(面向切面編程)，簡單舉的例子就能懂了，家裡出生了寶寶一枚，你需要每天餵奶、換尿布，某天爸媽剛好都沒空，於是找了保母，保母A只提供餵奶、換尿布的服務，重複性的物件，可以單獨分出就是AOP，提供服務的保母A就是DI，不信的保母A生病無法前來，我們換了方式找的保母公司，保母公司就是IOC，擁有管理多位保母，並提供服務，儘管保母B生病，公司會馬上找保母C提供服務。

DI,IOC主要就是說明不主動去請求，而是讓服務本身所提供，IOC又相較DI多容器做管理。

AOP則比較像是將公用方法，將重複性極高物件，單獨獨立的取出來。

Spring boot

Spring boot框架就是簡化配置，專案Maven透過POM檔進行配置即可，不用想SSH框架那樣複雜的配置，有缺專案就起步來，Spring boot就方便許多，但是就是要知道如何使用，汗。
